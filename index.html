Moduleとは

安全性・記述性・性能のバランスを追求したプログラミング言語です。Visual Studio Code (通称VS Code) などの環境構築を必要とせず、

  • 公式エディタ
  • コンパイラ

の二つのみで編集ができます。コンパイラは ここから 入手してください。

従来の課題Moduleの考え方
メモリ安全コンパイラが安全性を補助する
記述量必要最低限の構文で書ける
学習コスト初学者でも理解しやすい設計

コードの基本構成

基本的に

  1. fn initブロック内で宣言し、
  2. fn mainブロック内で実行したいことを書き、
  3. その間に定義を書く。

という流れです。コードを上部から読んだとき
宣言 → 定義 → 実行
となります (この順は必須でない) 。コンパイラは宣言を優先的に定義し、その次にユーザーの定義を最適化します。

はじめに

このパートでは、コードを書く際の基本となる情報を整理します。

コメント

コード中にメモを残しておきたいとき、率直に書くと定義されていない文字が含まれている可能性があるため、コンパイル時警告になります。安全にコメントを書く方法は以下の二通りあります。

構文

// コメント
/-
コメント
-/

実行したい内容と無関係な部分は、その部分をコメントにする工夫があります。

識別子と予約語

識別子

識別子とは、後述する予約語以外の、以下の規則がある語句のことです:

  • ユニコード文字である
  • 数字で始めてはいけない
  • 大文字と小文字は区別される

予約語

予約語は、既に使用が決まっている語句のことであり、以下の27個が挙げられます:

voidboolintfloatstr
setFILEERRORvarconst
deftefififdefand
ornottruefalseis
loopjumpimportofnew
in

式と文

式は命令を含まないもの、文は命令を含むものと定義します。文は改行区切りであり、他言語にみられる行末セミコロンは必要ありません。また、インデントはコンパイル時に読み飛ばされるので、プログラマーは自由にタブをいくつか置くことができます。

変数の宣言

var name : type = value

値20を、(符号付き)整数型で宣言します。

var age : int = 20
